# Zindan — инструкция для пользователя

**Версия:** 1.5.3 (сборка 235)  
**Исходники:** [github.com/GiorgioVik/zindan](https://github.com/GiorgioVik/zindan)

---

## Преамбула: зачем нужен Zindan

Zindan — это не «ещё одно приложение в списке», а средство цифровой гигиены, **инструмент изоляции** на базе **рабочего профиля Android**: отдельного пользователя в системе с собственными приложениями, данными и ярлыками (иконки с синим портфелем).

**Зачем это нужно:** до недавнего времени в **основном (личном) профиле** у вас обычно были установлены программы, которые должны быть всегда под рукой, но они ожидаемо **анализируют устройство и поведение** пользователя, чтобы быть максимально удобными для него — не обязательно вредоносные в классическом смысле, но собирающие много сигналов:

- **Государственные и окологосударственные сервисы** (идентификация, госуслуги, цифровые документы и т.п.);
- **Банки и платёжные приложения** с жёсткой проверкой окружения;
- **Крупные экосистемы** (почта, мессенджеры, маркетплейсы, «умный дом»), которые строят профиль пользователя и ограничивают доступ;
- **Приложения с агрессивной аналитикой**, рекламными SDK, антифрод-модулями;
- всё, что видит **установленные пакеты, сеть, VPN, контакты, файлы** в личном профиле.

Такие приложения **не должны** иметь доступ к другим чувствительным программам, которые вы запускаете для личных нужд, приватной переписки и работы, для доступа к AI приложениям с использованием VPN-клиента и т.д. В этой ситуации логично поместить «неблагонадёжные» приложения в изоляцию (в зиндан) и выпускать их оттуда, строго для выполнения ими заявленных функций, не кормя их ненужной им информацией и не давая повода блокировать вас.

**Zindan решает это так:**

1. **Клонирует** нужные приложения в **рабочий профиль** — они живут отдельно от личного и общаются с внешним миром строго по необходимости.
2. **Замораживает** их, когда вы ими не пользуетесь — они не крутятся в фоне и не «подсматривают» в сеть.
3. **Применяет режим Anti Spy** (включён всегда после настройки), который чувствует **VPN в основном (личном) профиле** и **автоматически замораживает** приложения из списка автозаморозки, чтобы изолированные программы не работали, пока активен туннель.

Zindan **не заменяет** второй телефон и **не блокирует** сами "сомнительные" приложения в личном профиле — он **отделяет** то, что вы сознательно держите в «изоляции» в рабочем профиле.

Zindan не предлагается для международного использования в виду специфической ситуации, поэтому никакие инструкции не переводились с русского языка на данном этапе разработки.

## 1. Первый запуск

1. Установите Zindan только в **основном** (личном) профиле.
2. Пройдите **мастер настройки** Zindan до конца.
3. В **системном** мастере Android подтвердите создание **рабочего профиля** и назначение Zindan **менеджером этого профиля** (администратор устройства внутри work profile — без этого изоляция не работает).
4. На **Android 7 и ниже** после создания профиля обязательно нажмите **уведомление** «Завершить настройку Zindan» (во время установки не включайте «Не беспокоить»). На Android 8+ этот шаг обычно не нужен.
5. После настройки откройте Zindan в основном профиле и **разрешите всё, что система запросит** (см. таблицу ниже).

### Разрешения при первом запуске


| Когда появляется                                          | Разрешение                                                                | Где дать                                | Зачем                                                               |
| --------------------------------------------------------- | ------------------------------------------------------------------------- | --------------------------------------- | ------------------------------------------------------------------- |
| Мастер Android при создании профиля                       | **Рабочий профиль** + **менеджер профиля** (администратор в work profile) | Системные экраны настройки              | Создание изолированного профиля и управление приложениями в нём     |
| Первое открытие главного экрана (Android 13+)             | **Уведомления**                                                           | Основной профиль → Zindan               | Служебные уведомления, в т.ч. «Контроль VPN» (Anti Spy)             |
| Первое подключение к рабочему профилю (Android 13+)       | **Уведомления**                                                           | Рабочий профиль → Zindan (вторая копия) | Фоновая служба в work profile                                       |
| Первый запуск приложения из work profile при активном VPN | **VPN** (доступ к VPN-соединению)                                         | **Основной** профиль → Zindan           | Кратко снять сторонний VPN перед запуском изолированного приложения |
| Samsung и часть прошивок при диалогах поверх лаунчера     | **Поверх других приложений** / всплывающие окна                           | Основной профиль → Zindan               | Чтобы системные диалоги Zindan не терялись за другими окнами        |


**Не при первом запуске**, а когда вы сами включаете функции в настройках (часто нужно дать **в обоих профилях** — личном и рабочем, если система показывает две копии Zindan):


| Функция в настройках                               | Разрешение                                                                   |
| -------------------------------------------------- | ---------------------------------------------------------------------------- |
| **Перенос файлов** (File Shuttle)                  | **Доступ ко всем файлам**; на Android 10+ также **поверх других приложений** |
| **Пропускать активные приложения** (автозаморозка) | **Доступ к истории использования** (статистика приложений)                   |
| **Установить APK в Zindan** (меню)                 | **Установка из неизвестных источников** для Zindan в рабочем профиле; при **активном VPN** в личном профиле пункт недоступен — сначала отключите VPN |


> Zindan опирается на **рабочий профиль Android**. На части прошивок (например, MIUI) функция может работать неполноценно.

### Очистка памяти — не удаляйте данные

Время от времени система Android или сам Zindan может предлагать **освободить память** (очистить кэш, оптимизировать хранилище и т.п.).

> **Внимание!** Ни в коем случае **нельзя удалять данные** приложения Zindan («Очистить данные», «Удалить данные», *Clear storage*) — **ни в основном, ни в рабочем профиле**. Иначе вы **потеряете доступ к рабочему профилю** и к изолированным приложениям; восстановить связь без удаления work profile и повторной настройки обычно **нельзя**.
>
> Допустимо: очистка **кэша**, если система предлагает только его. Недопустимо: **полная очистка данных** Zindan в любом профиле.

## 2. Главный экран приложения

Две вкладки внизу:


| Вкладка      | Содержимое                                                              |
| ------------ | ----------------------------------------------------------------------- |
| **Основной** | Приложения личного профиля — отсюда **клонируют** в Zindan              |
| **Рабочий**  | Приложения в изолированном профиле — здесь **замораживают** и запускают |


**Панель сверху (рабочий профиль):**

- 🔍 **Поиск** по приложениям
- ❄️ **Групповая заморозка** — заморозить все из списка автозаморозки
- 🧊 **Групповая разморозка** — разморозить все из списка
- ⚙️ **Настройки**

В меню (⋮): ярлыки на рабочий стол, установка APK, проводник файлов, «Показать все приложения».

Замороженные приложения в списке **вверху** и помечены как «[Заморожено]».

Приложения из **списка автозаморозки** отмечены **маленькой снежинкой** справа в строке списка.

## 3. Клонирование приложений

**Основной профиль** → долгое нажатие на приложение → **Клонировать в Zindan**.

После клонирования приложение появляется во вкладке **Рабочий**. Можно установить **вторую копию** того же приложения (личная + рабочая).

Дополнительно:

- **Установить APK в Zindan** (меню) — установка файла `.apk` прямо в рабочий профиль;
- **RuStore и другие магазины в work profile** — приложение ставится в рабочий профиль напрямую; Zindan автоматически добавляет его в **автозаморозку** (снежинка) и обновляет список, пока Zindan открыт (обычно в течение нескольких секунд, без переключения вкладок);
- обычные и пользовательские приложения клонируются через **установку APK** в целевой профиль;
- **системные** приложения клонируются иначе — через **`enableSystemApp()`** (включение уже вшитого в прошивку пакета в work profile), без отдельной установки.

### Системные приложения и неурегулированные сбои клонирования

Ниже — **проверенные на устройствах** ограничения, для которых **пока нет исправления** в Zindan (часть унаследована от upstream Shelter). Подробный журнал: [PROBLEMS.md](PROBLEMS.md).


| Проблема                                             | Статус                 | Симптом                                                                                                                                            | Почему так (технически)                                                                                                                                                                                                                                                                      | Обход                                                                                                                    |
| ---------------------------------------------------- | ---------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------ |
| **Galaxy Store** (`com.sec.android.app.samsungapps`) | ❌ отложено             | «Клонировать в Zindan» не срабатывает: тихий отказ или сообщение о **системном** приложении. Play Store и обычные приложения клонируются нормально | Пакет помечен как **system**, в work profile ещё **не установлен**. Zindan вызывает только `enableSystemApp()` — для OEM-магазина Samsung этого **недостаточно** (нужна полноценная установка, как у обычного APK; экспериментальный путь через PackageInstaller **откатан** — вызывал краш) | Установить нужные приложения через **Play Store** в work profile или **«Установить APK в Zindan»**                       |
| **Системное приложение → основной профиль**          | ⚠️ ограничение Android | Сообщение: *«Невозможно клонировать системные приложения в профиль, который Zindan не контролирует»*                                               | Клонирование **из work в личный** профиль для system-пакетов требует прав менеджера **в том** профиле. Zindan — менеджер только **work profile**, не личного                                                                                                                                 | Клонировать **из личного в work** (долгое нажатие во вкладке «Основной») или ставить APK вручную                         |
| **MIUI / HyperOS**                                   | ⚠️ прошивка            | Предупреждение при клонировании **несистемных** приложений; клон может не пройти                                                                   | Производитель ломает или ограничивает work profile                                                                                                                                                                                                                                           | Сначала клонировать **Play Store** (или системный магазин) в work profile, остальное ставить **из магазина внутри work** |
| **Скрытые системные приложения**                     | ⚠️ прошивка            | В списке work profile **нет** нужного системного приложения                                                                                        | Часть ROM не включает все system-пакеты в work profile по умолчанию                                                                                                                                                                                                                          | Меню ⋮ → **«Показать все приложения»** (осторожно: скрытые пакеты можно сломать) → попробовать включить вручную          |


**Что уже работает (для ориентира):** Google Play Store, обычные пользовательские приложения, ручная установка APK в work profile — на тестовых Samsung S24 Ultra и Tab S9 FE+ (см. чеклист в [PROBLEMS.md](PROBLEMS.md)).

**Что планируется, но ещё не сделано:** общее решение «system + пакет ещё не в профиле» без хардкода Samsung; повторный разбор `logcat`, если снова будет краш при установке через PackageInstaller.

## 4. Заморозка и разморозка одного приложения

Во вкладке **Рабочий** → нажатие на приложение:


| Действие          | Что делает                                                    |
| ----------------- | ------------------------------------------------------------- |
| **Заморозить**    | Скрывает приложение из лаунчера, останавливает фоновую работу |
| **Разморозить**   | Снова делает доступным, не запуская                           |
| **Запустить**     | Разморозка (если нужно) и запуск приложения                   |
| **Автозаморозка** | Добавить/убрать из списка групповой заморозки                 |
| **Ярлык запуска** | Иконка на рабочем столе для быстрого запуска                  |


**Заморозка** доступна только во вкладке **Рабочий**.

### Дополнительные пункты меню (короткий тап, только вкладка «Рабочий»)

Кроме заморозки и запуска, в меню есть два пункта из оригинального Shelter — это **не отдельная логика Zindan**, а тонкая настройка **стандартных API Android** для рабочего профиля. Zindan выступает **менеджером профиля** и добавляет или убирает **конкретный пакет** из системного белого списка. По умолчанию оба пункта **выключены**; при клонировании приложения они **не включаются** автоматически.


| Пункт меню                           | Технически (Android)                                                     | Что даёт на практике                                                                                                                                                                                                                          | Когда включать                                                                                         |
| ------------------------------------ | ------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ |
| **Виджеты в осн. профиле**           | `DevicePolicyManager.addCrossProfileWidgetProvider()`                    | Виджеты этого work-приложения можно **добавить на рабочий стол личного профиля**. Сам виджет работает в work profile (данные и учётка из «сейфа»), но отображается на домашнем экране основного пользователя; обычно помечен значком портфеля | Нужен виджет изолированного приложения (календарь, заметки, почта и т.п.) на личном рабочем столе      |
| **Разрешить работу между профилями** | `DevicePolicyManager.setCrossProfilePackages()` (только **Android 11+**) | Пакету **разрешено запросить у вас** право `INTERACT_ACROSS_PROFILES` — связь с **другой копией того же приложения** в личном профиле (если разработчик это заложил в приложение)                                                             | Приложение умеет cross-profile и вам нужен такой обмен; для обычной изоляции в Zindan **не требуется** |


**Важно:**

- Zindan **не включает** «работу между профилями» сам — только снимает административный запрет «этому пакету нельзя даже спросить». Дальше приложение (при поддержке в коде) может показать **свой** запрос согласия пользователю.
- Это **не то же самое**, что служебные «мосты» самого Zindan (запуск, заморозка, перенос файлов): они настраиваются Zindan при создании профиля и **не зависят** от этих переключателей.
- Отдельно Zindan по умолчанию разрешает из work profile открывать **ссылки в браузере** личного профиля и действие **«Поделиться»** — это другие системные правила, не этот пункт меню.
- Для модели угроз «сейф в work profile vs аналитика в личном» оба пункта **лучше оставлять выключенными**, если нет явной необходимости: они **точечно ослабляют изоляцию** выбранного приложения.

## 5. Список автозаморозки и групповые операции

**Список автозаморозки** — приложения, которые замораживаются **вместе** (групповые кнопки, ярлыки, Anti Spy, блокировка экрана).

Добавление: меню приложения → **Автозаморозка** (галочка в списке; справа появится снежинка).

**Групповая заморозка** — замораживает **все** из этого списка.  
**Групповая разморозка** — размораживает **все** из списка.

То же есть в **Настройки → Службы**.

## 6. Ярлыки на рабочем столе

В меню или в настройках можно создать:

## 7. Anti Spy (контроль VPN)

**Anti Spy включён всегда** после завершения настройки Zindan. Отдельного переключателя в интерфейсе нет.

### Что делает Anti Spy


| Событие                                                | Поведение                                                                                                                                        |
| ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Запуск Zindan** (после обновления или перезагрузки)  | Однократная **групповая заморозка** списка автозаморозки                                                                                         |
| **VPN включён** в основном профиле                     | **Автоматическая групповая заморозка** в рабочем профиле — без диалога «Да/Нет». Zindan **не отключает** ваш VPN                                 |
| **Запуск приложения** из work profile при активном VPN | Zindan пытается **кратко снять сторонний VPN** через служебный VPN-профиль (нужно разрешение VPN в основном профиле), затем запускает приложение |
| **VPN выключен**                                       | Сессия мониторинга сбрасывается; при следующем включении VPN заморозка снова выполнится                                                          |


### Служебное уведомление

В шторке остаётся уведомление **«Anti Spy — Контроль VPN»** — это нормально: фоновая служба следит за состоянием VPN, пока Zindan установлен и профиль настроен.

Дополнительно при подключении/отключении VPN могут появляться короткие сообщения **«VPN подключен»** / **«VPN отключен»**.

### Если приложение не заморозилось

При включённом VPN приложение на **переднем плане** (например, VPN-клиент в work profile) Android **не даёт** его скрыть, пока оно активно. Zindan **повторяет попытки** каждые несколько секунд, пока VPN активен. Закройте приложение вручную или переключитесь на другой экран — тогда оно попадёт в заморозку.

### Разрешения для Anti Spy

Один раз дайте:

- **VPN** для Zindan (основной профиль) — для снятия VPN перед запуском из work profile;
- **Уведомления** (Android 13+);
- при необходимости на Samsung — **всплывающие окна / поверх других приложений**.

## 8. Автозаморозка при блокировке экрана

**Настройки → Службы → Служба автозаморозки**

После запуска через **«Запустить»** или **ярлык запуска**, при **блокировке экрана** приложение снова заморозится.

Дополнительно:

- **Задержка автозаморозки** — через сколько заморозить после блокировки;
- **Пропускать активные приложения** — не замораживать то, что сейчас на экране (удобно для плеера; потом заморозьте вручную через **Групповую заморозку**).

## 10. Типичные сценарии

**Банк / мессенджер в изоляции**  
Клонировать в Zindan → добавить в автозаморозку → пользоваться через **Запустить** или **ярлык запуска** → после работы **Групповая заморозка** или блокировка экрана.

**Защита при включении VPN в личном профиле**  
Список автозаморозки заполнен → при включении VPN приложения из списка **замораживаются автоматически** (Anti Spy).

**Два аккаунта одного приложения**  
Одна копия в основном профиле, вторая (с синим портфелем) — в рабочем через Zindan.

**Установка из RuStore в рабочий профиль**  
Откройте RuStore в work profile, установите приложение. Вернитесь в Zindan (вкладка **Рабочий**): приложение появится в списке с **снежинкой** автозаморозки. Переключать вкладки не нужно, если Zindan уже открыт.

**Быстрый доступ**  
**Ярлык запуска** на рабочий стол рабочего профиля.

## 11. Удаление Zindan

> Не путайте **удаление Zindan** с **очисткой данных** в настройках приложения — см. §1 (*Очистка памяти — не удаляйте данные*).

1. **Настройки Android → Пользователи и аккаунты** — удалить **рабочий профиль** (Zindan).
2. Затем удалить приложение Zindan из основного профиля как обычно.

Удаление только иконки Zindan **не** удаляет рабочий профиль и клонированные приложения.

## 12. Ограничения (кратко)

- **Приложение на переднем плане** при VPN может оставаться видимым, пока вы его не закроете — см. §7.
- **Клонирование системных приложений** — см. подраздел в §3 (*Galaxy Store*, MIUI, system → main profile).
- Второй ярлык с **синим портфелем** на рабочем столе — нормальное поведение Android для work profile; при заморозке исчезает.
- Anti Spy **не отключает** сторонний VPN — только замораживает изолированные приложения и при запуске пытается временно снять VPN.

Подробнее об известных проблемах: [PROBLEMS.md](PROBLEMS.md).
